The rule.

A flex layout has two parts: one container and the items inside it.

container ← display: flex goes HERE
1
2
3
↑ items
HTML
<div class="container"> <div class="item">1</div> <div class="item">2</div> <div class="item">3</div> </div>
CSS
.container { display: flex; justify-content: space-between; align-items: center; gap: 16px; }

The class="container" in the HTML is what the .container rule in the CSS targets. The class name is the bridge.

This is the whole lesson in one slide.
  • Walk to the screen. Point at the orange box. Say: "This is the container. It's the parent element in the HTML — look at the <div class='container'> on the left."
  • Point at the green boxes. "These are the items. They're the children inside the container — the three <div class='item'> tags."
  • Point at the CSS. "The four properties are on the .container rule. They go on the container's CSS, not on the items. Notice there's no CSS for .item at all — the items don't need it. Their layout is decided by the container they live in."
  • The class-name bridge: trace your finger from class="container" in HTML to .container in CSS. "These have to match. If you rename one and forget the other, the styling breaks." This is worth labouring — class-name typos will be a frequent bug.
  • Comprehension check: "If I want to add a fourth item, what do I change?" (Answer: add another <div class='item'> in the HTML. Don't touch the CSS.) "And if I want them vertical instead of horizontal?" (Answer: add flex-direction: column to the .container rule.)
  • Don't move on until this lands. Pause. Look around. Anyone confused — fix it now.

An item can also be a container.

This is the bit that tripped us up. The links wrapper is an item of the nav, AND it's a container holding its own items.

container
item
item AND container
nav.main-nav ← container ├── a.brand ← item └── div.nav-links ← item AND container ├── a (Home) ← items inside ├── a (About) the nav-links ├── a (Gallery) container ├── a (Blog) └── a (Contact)
The big reveal of the lesson.
  • Slow down on this slide. This is the concept that didn't land last time.
  • Trace through the tree out loud: "The nav is a container. Inside it are TWO items — the brand on the left, and the links wrapper on the right. We need that wrapper so flexbox can push it to the end of the row."
  • Then: "But the links wrapper ITSELF is a flex container. Inside it are five link items. So we end up with two layers of flexbox: outer nav (`space-between` puts brand left, wrapper right), inner wrapper (arranges the five links in a row with gaps)."
  • Most common student mistake from last lesson: they put all 5 links and the brand directly inside `<nav>`, no wrapper. The nav becomes a container with 6 items, all spread evenly across the row. The brand floats in the middle, not on the left. That's the bug.
  • Comprehension check: "Why do we need the `<div class='nav-links'>` wrapper at all? Why not just put the links straight inside the nav?" (Looking for: because we want flexbox to treat the 5 links as ONE group, so the nav has just 2 items — brand and group — and `space-between` works.)

Same pattern. Three sections.

Build it once, you can build it everywhere. Today's job: do this same pattern for the cards and the footer.

NAV (done)
nav.main-nav ├── a.brand └── div.nav-links ├── a ├── a └── ...
CARDS (today)
section.cards ├── article.card ├── article.card └── article.card
FOOTER (today)
footer.site-footer ├── p (copyright) └── div.footer-links ├── a └── a

Notice: the nav and footer have the same shape. One element on the left, a wrapper holding items on the right.

The "ah-ha" slide.
  • Let them look at all three trees for a moment before you talk.
  • Then point out: nav and footer have identical shape. The cards section is simpler — just a container with three items, no nested wrapper needed.
  • Say: "You've already built the hardest one — the nav. The cards section is actually simpler. The footer is the nav pattern again."
  • Comprehension check: "Looking at the cards tree — where does `display: flex` go?" (On `.cards`.) "And what do I need on each `.card` to make them share space equally?" (`flex: 1`.)
  • This is where motivation lives. The work ahead is repetition of something they've already done. Tell them that.

A quick CSS reminder.

Before the next slide, you need to know what ".main-nav a" means versus ".main-nav". The space matters.

.main-nav
nav.main-nav ← THIS ONE ├── a.brand └── div.nav-links ├── a ├── a └── ...

Targets the nav element itself. One thing.

.main-nav a
nav.main-nav ├── a.brand ← all of └── div.nav-links these. ├── a ├── a └── ...

The space means "inside". Targets every <a> that lives inside .main-nav. Six things.

Different selectors target different elements. Pick the wrong one and your CSS lands on the wrong thing.

The bridge slide to the next comparison.
  • You can do this in 60 seconds if the class already gets descendant selectors — but most of yours won't, so slow down.
  • Trace through the left tree: ".main-nav on its own — no space, nothing after it. CSS reads this as 'find the element with class main-nav.' That's the <nav> itself. Just one thing."
  • Now the right tree: "Same start, but a space and then an a. The space is the magic. It means 'find every <a> tag that's INSIDE an element with class main-nav.' That's all six anchors — brand plus five links."
  • Comprehension check: "If I wrote .main-nav div, what would that target?" (Looking for: the div.nav-links wrapper. It's the only div inside.)
  • Set up the next slide: "On the next slide you'll see both these selectors used to apply display: flex. One works, one doesn't. Now you have the vocabulary to tell why."

Where the four properties go.

Same HTML, two different stylesheets. One works, one doesn't. Spot the difference.

✗ Wrong
.main-nav a { display: flex; justify-content: space-between; }

Flexbox is on each link. But links have nothing inside them to lay out. Nothing changes. The nav stays vertical.

✓ Right
.main-nav { display: flex; justify-content: space-between; }

Flexbox is on the nav, which contains the items. The nav arranges its children. Layout works.

The flex container is the parent. Always.

Lock-in slide.
  • This is the one to leave on the screen during independent work. Students who get stuck can glance up and check.
  • If anyone wrote the wrong version last lesson, they'll recognise it. That's the point.
  • Say: "When you write `display: flex`, the very next thing you should ask yourself is: what is this element the parent of? Those children are what's going to be laid out. If the answer is 'nothing,' you've put it on the wrong element."
  • Transition to independent work: "Open your starter — the new version with the structure trees in the comments. Build the cards section first. The pattern is on slide 5."
